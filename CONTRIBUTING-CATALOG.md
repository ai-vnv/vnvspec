# Contributing a Catalog Module

This document defines the inclusion policy for vnvspec best-practices catalogs.
A catalog module is acceptable for inclusion only if **all six** criteria are met.

## 1. Source Authority

Every requirement cites at least one of:

- Official package documentation (specific URL with anchor, not just homepage)
- Accepted RFC, ISO/IEC/IEEE/ANSI/UL standard
- OWASP guidance (ASVS, Top 10, Cheat Sheet Series)
- NIST publication (SP 800-series, AI RMF)
- Peer-reviewed paper (with DOI)
- Recognized industry methodology (e.g., 12-factor app, Karpathy's NN training recipe)

The citation lives in the `Requirement.source` field.

## 2. Named Maintainer

A human (or a named lab account mapping to a human) commits to:

- Reviewing the catalog every 6 months for drift against the upstream package
- Responding to issues filed against the catalog within 30 days
- Triggering a catalog version bump when the upstream package releases an incompatible major version

## 3. Version Pin

Each catalog module declares the package version range it was authored against:

```python
__compatible_with__ = "torch>=2.3,<3.0"
```

## 4. Honest Scope Statement

Every catalog module's `__doc__` opens with exactly:

> *This catalog reflects published best practices for [PACKAGE] as of [YYYY-MM-DD]. It is a baseline, not a substitute for expert review.*

## 5. Standards Mappings Where Applicable

- Web security catalogs map to OWASP ASVS
- ML training catalogs map to NIST AI RMF
- Medical/safety catalogs map to relevant ISO clauses
- Absence of a standards mapping is noted in the module docstring

## 6. Tests for the Catalog Itself

Auto-generated by `tests/catalog/conftest.py::validate_catalog_requirement()`:

- Has `source` populated with at least one URL
- Validates under GtWR formal profile with zero error-severity violations
- Round-trips through JSON and dict serialization
- ID matches `CAT-<PKG>-<TOPIC>-<NNN>`
- `priority` is valid

## ID Convention

`CAT-<PKG3>-<TOPIC4>-<NNN>` where:

- `PKG3` = 2-5 letter package abbreviation (e.g., PYT, HGF, FPI, SQA, PYO)
- `TOPIC4` = 2-6 letter topic abbreviation (e.g., REPRO, GRAD, CHKPT)
- `NNN` = 3-digit sequence number

IDs are stable forever once shipped; deprecated, never renumbered.

## Module Layout

```
src/vnvspec/catalog/
    _base.py                      # Infrastructure (do not modify)
    <domain>/
        <package>/
            __init__.py
            <topic>.py            # Each topic returns list[Requirement]
```

## Catalog Function Convention

A catalog function returns `list[Requirement]`. Pure function, stable IDs across releases.

```python
def reproducibility() -> list[Requirement]:
    """PyTorch reproducibility best practices."""
    return [
        Requirement(
            id="CAT-PYT-REPRO-001",
            statement="The training script shall set ...",
            source=["https://docs.pytorch.org/docs/stable/notes/randomness.html"],
            priority="blocking",
            verification_method="test",
            ...
        ),
    ]
```
